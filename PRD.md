# Resume Coach — Product Requirements Document

## Problem

Tech professionals treat resume writing as a fire drill. They scramble to remember years of accomplishments when they suddenly need a job, then copy-paste the same generic resume to every application. The result: weak resumes that don't land interviews, and hours wasted per application trying to tailor them.

Meanwhile, AI writing tools hallucinate — they invent accomplishments, inflate numbers, and produce resumes that fall apart in interviews when candidates can't back up what's on the page.

## Vision

Resume Coach is the place where tech professionals build their career story over time and deploy it on demand. Users continuously log accomplishments as they happen — not months later from foggy memory — and when a job opportunity appears, the system crafts a tailored resume grounded entirely in what the user actually did. No hallucinations. No generic templates. Every bullet point traces back to a real accomplishment.

The key insight: **the best time to write your resume is every month, not when you're job hunting.**

## Target User

Tech professionals — software engineers, data scientists, product managers, designers, engineering managers — who:
- Want to land roles at specific companies, not spray-and-apply
- Value accuracy and authenticity over keyword stuffing
- Are too busy to maintain polished resumes while employed
- Need help translating raw "what I did" into compelling "what I achieved"

## Core Concepts

### Accomplishment Profile (Source of Truth)
A living document of the user's career accomplishments, organized by job/role. Raw, honest notes — not polished resume language. This is never sent to employers directly. It's the quarry from which resumes are carved.

### Accomplishment Tracking (Continuous)
Users are prompted on a recurring schedule (default monthly, user-configurable frequency) to log what they've done recently. Low friction — raw notes are fine. The system helps them structure and flesh out entries over time. This habit means the profile is always current, not reconstructed from memory.

### Performance Review Assistant
When review season comes, users share their company's perf review format and the system drafts a review pulling from the accomplishment profile. This is the "value before job seeking" hook — users get immediate utility from logging accomplishments, long before they need a resume.

### Job Opportunity
A job the user is interested in. The system scrapes the posting, saves it as inspectable/editable markdown, and researches the company culture — especially within the function the role sits in. This context informs how the resume is tailored.

### Tailored Resume
Generated by selecting and rephrasing accomplishments from the profile to match what a specific role values. The system picks, prioritizes, and phrases — but never invents. Every bullet must trace to a real accomplishment in the profile.

### Cover Letter
Generated alongside the tailored resume using the same grounded approach — profile accomplishments + job context + company research.

### Job Tracker
Lightweight kanban for tracking application status (saved, applied, interviewing, offer, rejected, withdrawn). Notes field per job for user context.

## User Journeys

### Journey 1: "I just want to keep track of what I do" (Continuous)
1. User signs up, adds current and past roles to their profile
2. Logs raw accomplishments under each role ("led migration to k8s, reduced deploy time from 45min to 8min")
3. Receives monthly reminder to log recent work
4. Over time, builds a rich accomplishment history without pressure

### Journey 2: "Review season is coming" (Periodic)
1. User uploads or describes their company's perf review format
2. System drafts a review pulling from recent accomplishments
3. User edits and refines — the draft is a starting point, not a final product
4. Accomplishments used in the review are marked/tagged for reference

### Journey 3: "I found a role I want to apply for" (On-demand)
1. User pastes a job posting URL
2. System scrapes the posting, saves structured markdown, researches the company
3. User reviews the scraped posting and company research, edits if needed
4. System generates a tailored resume by selecting and rephrasing profile accomplishments
5. User reviews, adjusts through chat or direct editing
6. System generates a matching cover letter
7. User exports as PDF or ATS-friendly plain text
8. Job moves to "applied" on the tracker

### Journey 4: "I need to track my applications" (Ongoing)
1. User moves jobs through kanban stages
2. Adds notes per job (interview feedback, contacts, follow-up dates)
3. Has a clear view of where every application stands

## Feature Requirements

### P0 — Core (Must Have)

**Accomplishment Profile**
- Create/edit profile with roles organized by employer and position
- Free-text accomplishment entry per role (markdown supported)
- AI assistant helps refine raw notes into structured accomplishments without changing the facts

**Job Import & Research**
- Paste URL to import job posting
- Scrape and save as editable markdown
- Web search for company culture and function-specific context
- All research visible and editable by user

**Tailored Resume Generation**
- AI selects relevant accomplishments from profile for a given job
- Rephrases for impact and relevance — never fabricates
- User can accept, reject, or modify each suggestion
- Professional summary tailored to the role
- Skills section derived from actual experience

**Resume Editing & Export**
- Direct editing of all resume sections
- Chat-based AI coaching for refinement
- Export as PDF (print-ready)
- Export as plain text / markdown (ATS-friendly)

**Authentication & Basics**
- Email/password and Google OAuth sign-in
- User account management

### P1 — Important (Should Have)

**Accomplishment Reminders**
- Recurring prompts to log accomplishments (default monthly)
- User-configurable frequency (weekly, biweekly, monthly, quarterly)
- Low-friction entry — quick note capture, refine later

**Cover Letter Generation**
- Generated from profile + job context + company research
- Same no-hallucination constraint as resumes
- Editable and exportable

**Job Tracker**
- Kanban board with stages: saved, applied, interviewing, offer, rejected, withdrawn
- Notes field per job
- Link between job and its tailored resume

**Performance Review Assistant**
- User provides their company's review format/template
- System drafts review from recent accomplishments
- Editable output

### P2 — Nice to Have (Could Have)

**LinkedIn Import**
- Import work history from LinkedIn to bootstrap the profile
- User reviews and enriches imported data

**Resume Versioning**
- Track changes across resume iterations
- Compare versions side-by-side

**Analytics**
- Application funnel metrics (how many applied vs. interviews vs. offers)
- Which accomplishments get used most across tailored resumes

## Non-Functional Requirements

- **Accuracy above all**: The system must never generate accomplishments, metrics, or claims not grounded in the user's profile. When in doubt, leave it out.
- **Transparency**: Every AI-generated suggestion should be traceable to source accomplishments. Users must be able to inspect what the AI drew from.
- **Privacy**: User career data is sensitive. No training on user data. Clear data ownership.
- **Speed**: Resume generation should complete in under 30 seconds. Chat responses should stream in real-time.
- **Reliability**: Export formats must render correctly across ATS systems and PDF viewers.

## What's Already Built

The current codebase (Next.js 16, tRPC, Prisma/PostgreSQL, LangGraph + OpenAI) has:

- **Working**: Auth (email/password + Google OAuth), resume CRUD, resume editor with sections (experience, education, skills, summary, contact info), AI chat coaching per resume with streaming, job posting scrape & import, resume duplication, markdown/print export, resume list with sort/filter
- **Data models exist for**: Users, resumes, experiences, positions, skills, education, patents, jobs, chat threads/messages
- **Not yet built**: Accomplishment profile (separate from resumes), accomplishment reminders, company culture research, cover letter generation, tailored resume generation from profile, job tracker kanban, perf review assistant, PDF export, LinkedIn import

## Open Questions

1. **Profile vs. Resume data model** — Should the accomplishment profile be a distinct entity from resumes, or should resumes be "views" generated from a shared experience/accomplishment pool? (Leaning: distinct profile as source of truth, resumes as derived artifacts)
2. **Reminder delivery** — Email, in-app notification, or both? Push notifications if we go mobile later?
3. **Company research depth** — How much automated research is useful vs. noise? Should users be able to trigger deeper research on demand?
4. **ATS compatibility testing** — Should we validate exported resumes against common ATS parsers?
5. **Collaboration** — Any interest in resume review by peers/mentors in the future?
